Getting Started

AI Token King provides a unified API gateway to multiple AI models — GPT, Claude, Gemini, DeepSeek, and Qwen — through a single endpoint and token balance. This documentation covers everything you need to integrate AI Token King into your applications.

Authentication

All API requests require an API key passed as a Bearer token in the Authorization header. You can generate API keys from your dashboard under Settings → API Keys.

Authorization: Bearer atk_your_api_key_here

Never expose your API key in client-side code or public repositories. Use environment variables and server-side requests.

Your First Request

Here's a minimal example to get a completion from GPT-4o using the AI Token King API:

curl https://api.aitokenking.com/v1/chat/completions \ -H "Content-Type: application/json" \ -H "Authorization: Bearer atk_your_key" \ -d '{ "model": "gpt-4o", "messages": [ { "role": "user", "content": "Explain AI tokens in one sentence." } ] }'

The response follows the OpenAI-compatible format, making it easy to migrate existing integrations:

{ "id": "chatcmpl-abc123", "object": "chat.completion", "model": "gpt-4o", "choices": [{ "message": { "role": "assistant", "content": "AI tokens are the small text units that language models use to process and generate text, with usage charged per token consumed." }, "finish_reason": "stop" }], "usage": { "prompt_tokens": 12, "completion_tokens": 28, "total_tokens": 40 } }

API Reference

The AI Token King API is OpenAI-compatible. If you're already using the OpenAI SDK, you can switch to AI Token King by updating the base URL and API key.

Base URL & Headers

Base URL: https://api.aitokenking.com/v1 Required headers: Content-Type: application/json Authorization: Bearer <your_api_key>

Chat Completions

POST /v1/chat/completions

Creates a completion for the provided messages. Supports all models available on the platform.

Parameter Type Required Description
model string required Model ID (e.g. gpt-4o, claude-3-5-sonnet)
messages array required Array of message objects with role and content
max_tokens integer optional Maximum tokens to generate in the response
temperature float optional Sampling temperature 0–2. Higher = more creative, lower = more deterministic. Default: 1
stream boolean optional Stream partial responses via SSE. Default: false
system string optional System prompt to set model behavior and context

Models Endpoint

GET /v1/models — Returns a list of all available models and their current status.

curl https://api.aitokenking.com/v1/models \ -H "Authorization: Bearer atk_your_key"

Token Calculation & Pricing

AI Token King charges per token consumed across input and output. Tokens are deducted from your balance in real time. You can monitor usage from your dashboard.

How Tokens Are Counted

  • Input tokens: Every character you send — your messages, system prompt, and conversation history
  • Output tokens: Every character the model generates in response
  • Both directions count toward your usage
  • Longer conversations accumulate context tokens — older messages still count as input
Quick Reference
1,000 tokens ≈ 750 words
1 page (A4) ≈ 500–600 tokens
Short prompt ≈ 10–50 tokens
Full report ≈ 5,000–30,000 tokens

OpenClaw API

OpenClaw is AI Token King's AI Agent layer. Unlike single-turn completions, OpenClaw agents can execute multi-step workflows, maintain state across sessions, and trigger actions based on conditions.

OpenClaw Overview

  • Agents are defined by a workflow configuration — a JSON object describing steps, conditions, and actions
  • Each agent has a persistent session and can be triggered via webhook, schedule, or API call
  • Agents use your token balance — each step consumes tokens based on the model assigned to it

Creating an Agent

POST /v1/agents

{ "name": "Customer Support Bot", "model": "gpt-4o-mini", "system_prompt": "You are a helpful customer support agent for...", "steps": [ { "id": "intake", "type": "message", "trigger": "on_start" }, { "id": "escalate", "type": "webhook", "condition": "sentiment == 'negative'", "url": "https://your-app.com/escalate" } ], "schedule": null }

Once created, the agent returns an agent_id you can use to trigger runs, check status, and retrieve logs via the Agents API.

SDKs & Rate Limits

SDKs & Libraries

Because AI Token King uses an OpenAI-compatible API, you can use the official OpenAI SDKs by pointing them to our base URL:

# Python (OpenAI SDK) from openai import OpenAI client = OpenAI( api_key="atk_your_key", base_url="https://api.aitokenking.com/v1" ) response = client.chat.completions.create( model="gpt-4o", messages=[{"role": "user", "content": "Hello!"}] )
// Node.js (OpenAI SDK) import OpenAI from 'openai'; const client = new OpenAI({ apiKey: 'atk_your_key', baseURL: 'https://api.aitokenking.com/v1', }); const response = await client.chat.completions.create({ model: 'claude-3-5-sonnet', messages: [{ role: 'user', content: 'Hello!' }], });
